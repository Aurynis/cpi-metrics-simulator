Sobre este simulador

Este simulador aplica-se exclusivamente à capability Cloud Integration do SAP Integration Suite. Outras capabilities (API Management, Event Mesh, OData Provisioning) têm regras de metering diferentes. As regras de cálculo seguem o SAP Note 2942344 (v23, 17.06.2025) — "SAP Integration Suite - Message Metric Definition".

Limitation ativa (mar/2026): O SAP Note declara: "Message size measurement is not available as of now. Until it gets available, every integration flow execution with 1 or more receiver adapters will be counted as 1 message irrespective of message size." Isso significa que hoje 1 execução = 1 message, independente do payload. A coluna "futuro" projeta o cenário quando a SAP remover esta limitation e ativar o metering por tamanho (250KB por message). Não há data confirmada para esta ativação.

Regras de metering aplicadas

  • Regra base: Tamanho agregado de outbound via receiver adapter(s). Cada bloco de 250KB (ou fração) = 1 message. Request + response somam.
  • Splitter: Cada split outbound via receiver adapter é uma message separada (sujeita à regra de 250KB individualmente).
  • JMS / Process Direct: Mensagens via esses adapters são excluídas do metering — são internas ao tenant.
  • Retry: Cada retry via receiver adapter conta como message separada.
  • DEV/QAS/PRD: Todas as landscapes contam. Sem isenção para ambientes de teste.
  • SAP-to-SAP: Mensagens via packages SAP standard não editados são gratuitas.

Status da ferramenta de medição da SAP

Até março/2026, a SAP não publicou roadmap com data para ativação do size metering. O SAP Note 2942344 mantém a limitation como "until it gets available" desde sua primeira versão. Na comunidade, há pedidos ativos para publicação da API de metering (/rest/api/v1/metering/usage/) como endpoint oficial — hoje ela é interna. A funcionalidade de "Message Usage Analytics" no monitor do CPI (dez/2025) mostra breakdown por iFlow, mas ainda conta 1 execution = 1 message conforme a limitation.
